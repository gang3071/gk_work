## 🔴 发现的隐藏问题

### 问题1：DG 控制器的 diff 计算错误（严重）

**问题位置**: `app/wallet/controller/game/DGGameController.php:120`

**问题代码**:
```php
// type=2:派彩 type=5:红包 → Lua 原子结算
$luaParams = [
    'order_no' => $orderNo,
    'platform_id' => $this->service->platform->id,
    'amount' => $amount,
    'diff' => $amount,  // ❌ 错误：type=2 派彩应该是 win - bet
    'game_code' => $detail['gameId'] ?? '',
    'transaction_type' => $type == 5 ? TransactionType::SETTLE_REWARD : TransactionType::SETTLE,
    'original_data' => $params,
];
```

**问题分析**:
- **type=2 派彩**: `diff` 应该是 `win - bet`，而不是直接用 `amount`
- **type=5 红包**: 如果是额外奖励，`diff = amount` 可能是对的
- 当前逻辑对 type=2 和 type=5 统一处理，导致 type=2 的 diff 计算错误

**影响**:
- 派彩记录的 `diff` 字段不准确
- 玩家输赢统计错误
- 反水计算可能受影响

**修复优先级**: 🔴 高

**修复方案**:
```php
if (in_array($type, [2, 5])) {
    // ✅ 修复：区分派彩和红包的 diff 计算
    if ($type == 2) {
        // type=2 派彩：需要计算 diff = win - bet
        $betAmount = getBetAmountWithFallback('DG', $orderNo, $player->id);
        $diff = bcsub($amount, $betAmount, 2);
    } else {
        // type=5 红包：额外奖励，diff = amount
        $diff = $amount;
    }

    $luaParams = [
        'order_no' => $orderNo,
        'platform_id' => $this->service->platform->id,
        'amount' => $amount,
        'diff' => $diff,
        'game_code' => $detail['gameId'] ?? '',
        'transaction_type' => $type == 5 ? TransactionType::SETTLE_REWARD : TransactionType::SETTLE,
        'original_data' => $params,
    ];
    
    // ...
}
```

### 问题3：数据库查询性能问题（中等）

**问题位置**: `app/functions.php:2033-2040`

**问题代码**:
```php
$query = \app\model\PlayGameRecord::query()
    ->where('order_no', $orderNo)
    ->where('platform_id', function ($subQuery) use ($platform) {
        // ❌ 性能问题：每次都子查询 game_platform 表
        $subQuery->select('id')
            ->from('game_platform')
            ->where('code', $platform)
            ->limit(1);
    });
```

**问题分析**:
- 每次降级查询都会执行子查询获取 `platform_id`
- `game_platform` 表数据变化少，不需要每次查询
- 降级场景已经是性能瓶颈，子查询进一步降低性能

**影响**:
- 降级查询性能下降
- 数据库负载增加

**修复优先级**: 🟡 中

**修复方案A（简单）**: 使用缓存平台ID映射
```php
// 在文件顶部添加缓存函数
function getPlatformIdByCode(string $platform): ?int
{
    static $cache = [];
    
    if (!isset($cache[$platform])) {
        $cache[$platform] = \app\model\GamePlatform::query()
            ->where('code', $platform)
            ->value('id');
    }
    
    return $cache[$platform];
}

// 在 getBetAmountWithFallback 中使用
$platformId = getPlatformIdByCode($platform);
if ($platformId) {
    $query = \app\model\PlayGameRecord::query()
        ->where('order_no', $orderNo)
        ->where('platform_id', $platformId);  // ✅ 直接使用ID
}
```

**修复方案B（推荐）**: 调用方传入 platform_id
```php
// 修改函数签名
function getBetAmountWithFallback(
    string $platform, 
    string $orderNo, 
    ?int $playerId = null,
    ?int $platformId = null  // ✅ 新增参数
): float {
    // ...
    
    $query = \app\model\PlayGameRecord::query()
        ->where('order_no', $orderNo);
    
    if ($platformId !== null) {
        // ✅ 优先使用传入的 platform_id
        $query->where('platform_id', $platformId);
    } else {
        // 降级使用 platform code 查询
        $query->where('platform_id', function ($subQuery) use ($platform) {
            $subQuery->select('id')
                ->from('game_platform')
                ->where('code', $platform)
                ->limit(1);
        });
    }
    
    // ...
}

// 调用时传入 platform_id
$betAmount = getBetAmountWithFallback(
    'KT', 
    $orderNo, 
    $player->id,
    $this->service->platform->id  // ✅ 传入已知的 platform_id
);
```
