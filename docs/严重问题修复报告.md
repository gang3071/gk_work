## ✅ 问题1：缓存一致性风险（已有实现）

### 问题描述
非 Lua 操作（如后台手动调整余额、充值、提现）更新数据库后，Redis 缓存未同步，导致余额不一致。

### 修复方案
**发现代码中已实现自动缓存同步机制**，位于 `app/model/PlayerPlatformCash.php`：

```php
protected static function booted()
{
    static::updated(function (PlayerPlatformCash $wallet) {
        if (!$wallet->wasChanged('money')) {
            return;
        }

        // ✅ 自动同步 Redis 缓存
        $cacheUpdated = \app\service\WalletService::updateCache(
            $wallet->player_id,
            $wallet->platform_id,
            (float)$wallet->money
        );

        // 🚨 缓存同步失败告警
        if (!$cacheUpdated) {
            \support\Log::critical('PlayerPlatformCash: Redis cache sync failed!', [
                'player_id' => $wallet->player_id,
                // ...
            ]);
        }
    });
}
```

### 修复文件
- `app/model/PlayerPlatformCash.php` - **第 60-99 行**（已有实现，无需修改）

### 验证方法

## ✅ 问题3：diff 计算无降级方案

### 问题描述
结算时 diff 计算依赖 Redis 中的 bet 记录，如果 Redis 记录过期（7天 TTL）或 Redis 故障，会导致：
- `betAmount = 0`
- `diff = win - 0 = win`（错误）

### 修复方案
新增降级查询函数，优先从 Redis 获取，失败时从数据库查询。

#### 新增辅助函数

**文件**: `app/functions.php`

**函数1**: `getBetAmountWithFallback($platform, $orderNo, $playerId)`

```php
/**
 * 获取下注金额（带数据库降级）
 *
 * 1. 优先从 Redis 获取下注记录
 * 2. Redis 未命中，从数据库查询 PlayGameRecord
 * 3. 记录降级告警日志
 *
 * @param string $platform 平台代码 (KT, ATG, DG等)
 * @param string $orderNo 订单号
 * @param int|null $playerId 玩家ID
 * @return float 下注金额
 */
function getBetAmountWithFallback(string $platform, string $orderNo, ?int $playerId = null): float
{
    // 1. 从 Redis 获取
    $betRecordKey = "game:record:bet:{$platform}:{$orderNo}";
    $betRecord = \support\Redis::hGetAll($betRecordKey);

    if (!empty($betRecord) && isset($betRecord['amount'])) {
        return (float)$betRecord['amount'];
    }

    // 2. Redis 未命中，从数据库降级
    $dbRecord = \app\model\PlayGameRecord::query()
        ->where('order_no', $orderNo)
        ->where('platform_id', function ($subQuery) use ($platform) {
            $subQuery->select('id')
                ->from('game_platform')
                ->where('code', $platform)
                ->limit(1);
        })
        ->when($playerId, fn($q) => $q->where('player_id', $playerId))
        ->first(['bet']);

    if ($dbRecord) {
        // 🚨 记录降级告警
        \support\Log::warning('[降级] Redis bet记录缺失，已从数据库查询', [
            'platform' => $platform,
            'order_no' => $orderNo,
            'bet_amount' => $dbRecord->bet,
        ]);
        return (float)$dbRecord->bet;
    }

    // 3. 完全缺失
    \support\Log::error('[严重] bet记录完全缺失（Redis + DB）', [
        'platform' => $platform,
        'order_no' => $orderNo,
    ]);

    return 0;
}
```

## ✅ 问题5：退款金额未验证

### 问题描述
取消/退款时未验证退款金额是否合理，可能出现：
- 退款金额 > 原下注金额
- 数据异常或恶意请求

### 修复方案
新增退款金额验证函数，在执行 Lua 原子取消前进行检查。

#### 新增辅助函数

**文件**: `app/functions.php`

**函数2**: `validateRefundAmount($platform, $orderNo, $refundAmount, $playerId)`

```php
/**
 * 验证退款金额是否合理（带数据库降级）
 *
 * @param string $platform 平台代码
 * @param string $orderNo 订单号
 * @param float $refundAmount 退款金额
 * @param int|null $playerId 玩家ID
 * @return array ['valid' => bool, 'bet_amount' => float, 'message' => string]
 */
function validateRefundAmount(string $platform, string $orderNo, float $refundAmount, ?int $playerId = null): array
{
    // 获取原下注金额（使用降级方案）
    $betAmount = getBetAmountWithFallback($platform, $orderNo, $playerId);

    // 验证退款金额
    if ($refundAmount > $betAmount) {
        \support\Log::warning('[退款验证] 退款金额超过下注金额', [
            'platform' => $platform,
            'order_no' => $orderNo,
            'refund_amount' => $refundAmount,
            'bet_amount' => $betAmount,
            'excess' => $refundAmount - $betAmount,
        ]);

        return [
            'valid' => false,
            'bet_amount' => $betAmount,
            'message' => sprintf(
                '退款金额(%.2f)超过原下注金额(%.2f)',
                $refundAmount,
                $betAmount
            ),
        ];
    }

    return [
        'valid' => true,
        'bet_amount' => $betAmount,
        'message' => 'ok',
    ];
}
```

#### 修改 RedisLuaScripts::atomicCancel

```php
public static function atomicCancel(int $playerId, string $platform, array $data): array
{
    // 参数验证
    validateLuaScriptParams($data, [
        'order_no' => ['required', 'string'],
        'refund_amount' => ['required', 'numeric', 'min:0'],
        'transaction_type' => ['required', 'string'],
    ], 'atomicCancel');

    $orderNo = $data['order_no'];
    $refundAmount = $data['refund_amount'];

    // ✅ 问题5修复：验证退款金额是否合理
    $skipValidation = $data['skip_amount_validation'] ?? false;
    if (!$skipValidation && $refundAmount > 0) {
```
